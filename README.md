# Push

Production-ready artifacts for web deployment.

> **Only populate this folder if this project is being deployed to the web.**
> Leave it empty for scripts, internal tools, data work, or anything not going live online.

**Examples:**
- Built frontend (e.g. `dist/` output from Vite/Next.js)
- Static site output
- Dockerfiles and deployment configs
- Environment variable examples (`.env.example`)
- CI/CD pipeline configs
